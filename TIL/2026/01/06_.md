## 1. 먼저, 쓰시마 구조를 한 번에 조망하기 (1주)

### 할 일

1. **메인 / 사이드 구조 파악**

   * 메인 스토리(“Jin’s Journey”),
     **Tales of Tsushima(동료·NPC 중심 서브 퀘스트)**,
     Mythic Tales(전설 무기·기술)로 크게 나뉘는 구조를 정리.
   * 각 카테고리마다

     * “이 퀘스트의 목적은 뭔가?”
     * “레벨 구조(마을, 야영지, 숲, 절벽 등)는 어떤 패턴인가?”
       를 간단히 노트.

2. **지도 위에서 활동 종류 분류**

   * 폭스 덴(여우굴) / 이나리 신사
   * 대나무 도장(Bamboo Strike)
   * 온천 / 하이쿠 / 신토 신사 / Pillar of Honor 같은 포인트들
   * 몽골 야영지 / 요새 / 마을
   * “전투/잠입형”, “짧은 상호작용형”, “이동/플랫폼형”, “스토리형”으로 나눠서,
     각 타입마다 **레벨에서 요구하는 플레이 액션**(이동, 잠입, 조사, 대화 등)을 간단히 정리.

> 이 단계에서는 코드 생각 전혀 안 하고,
> “쓰시마가 어떤 활동들을 어떤 ‘블록’으로 반복해서 보여주는지”만 감각적으로 외워두는 느낌으로 가면 돼요.

## 3. 폭스 덴 & 이나리 신사 – 환경 유도로 목적지에 데려가기

### 3-1. 게임 안에서 할 관찰

폭스 덴을 발견하면 여우가 나타나서 **신사까지 길 안내**를 하고, 신사에 도착해 기도를 올리면 부적/슬롯이 강화되는 구조죠.

플레이할 때는:

1. **여우를 처음 발견하는 순간**

   * 카메라 각도, 여우의 위치, 주변 지형(언덕, 나무, 바위), 색 대비를 캡처/스크린샷.
   * “플레이어 시선이 여우에게 어떻게 끌리는지” 관찰.

2. **여우가 안내하는 동선**

   * 여우가 멈추는 지점마다

     * “여기서 플레이어가 뭘 보게 되는지(랜드마크, 신사 일부 등)” 기록.
   * 위험 지형(낭떠러지, 점프 구간), 좁은 길, 통로 역할 하는 나무/바위 등 체크.

3. **신사 도착 후**

   * 상호작용(기도) 트리거 위치, 카메라 연출, 피드백(진의 제스처, UI, 사운드)을 메모.
   * 보상(부적/슬롯 증가)이 언제·어떻게 보여지는지 기록.

이런 식으로 플레이하면서 **“기계적인 구조 + 레벨 구조 + 연출”**을 한꺼번에 뽑아내는 거예요.

참고로, 쓰시마의 시각적 유도/바람 시스템을 연구한 논문·GDC 세션이 있어서,
“왜 굳이 나침반 대신 바람·여우·새를 썼는지”를 같이 보면 설계 철학 이해에 도움됩니다.

### 3-2. 여기서 도출할 “액션 블록 목록”

폭스 덴 구조를 **재사용 가능한 블록들**로 쪼개보면:

* `BP_FoxGuide`

  * 시작 위치, 경유 지점(스플라인/웨이포인트) 배열을 가지는 AI
  * 플레이어가 특정 범위 안에 있으면 달리기 시작
  * 도중 중간중간 멈추며 플레이어를 기다림
* `BP_Shrine`

  * 일정 반경 안에 들어오면 인터랙트 팝업
  * 상호작용 시 애니메이션/카메라/보상 트리거
* `BP_RewardManager`

  * 신사 수를 관리해 일정 개수마다 능력치/슬롯 증가 적용

### 3-3. 언리얼/블루프린트 과제

1. **미니 지형 만들기**

   * 언덕 + 작은 숲길 + 신사가 보이는 지형을 간단한 블록/지형 툴로 제작.

2. **여우 가이드 블루프린트**

   * Spline Component 또는 여러 Target Point를 이용해
     여우가 미리 정한 경로를 따라 이동하도록 AI Move To 구현.
   * 여우가 일정 지점마다 잠깐 멈춰서 플레이어를 쳐다보도록(회전만 해도 됨).

3. **신사 상호작용**

   * 신사에 Interact 키(E 등)를 추가하고,
     인터랙션 시

     * 카메라를 신사 쪽으로 당기고
     * UI 텍스트(“부적 슬롯 +1” 같은 가짜 메시지) 표시.

4. **보상 시스템**

   * `CollectedShrineCount` 변수를 GameInstance / SaveGame 등에 저장하고,
     `n`개 이상이면 플레이어에게 능력치(예: 체력, 이동속도) 버프 주기.

> 목표:
> “쓰시마만큼 멋있게가 아니라, **동일한 구조의 미니 버전**이 UE에서 돌아가게 하기.”

## 4. 대나무 도장 – 입력 퍼즐 액션 블록

대나무 도장은 정해진 버튼 조합을 제한 시간 안에 성공하면 **Resolve(기력) 최대치가 늘어나는 미니 게임**이죠.

### 4-1. 관찰 포인트

* 시퀀스가 어떻게 표시되는지 (텍스트/아이콘 순서, 속도)
* 시간 제한 표시 방식 (UI, 사운드 압박)
* 실패/성공했을 때 연출 차이 (카메라, 애니메이션, UI)

### 4-2. 액션 블록 설계

* `BP_InputSequencePuzzle`

  * `Sequence` : 키 배열 (예: {E, F, Space})
  * `TimeLimit` : 제한 시간
  * `OnSuccess`, `OnFail` 이벤트 디스패처 (외부 레벨에서 연결)

블루프린트 과제:

* 랜덤 시퀀스를 생성하거나, 사전에 만든 3~5개 시퀀스 중 하나를 사용.
* 입력 하나 받을 때마다 UI 상에서 “진행 상태” 업데이트.
* 성공 시 ResolveBarMax 변수 증가, 실패 시 리셋.

이 블록은 나중에 **다른 퍼즐에도 재활용** 가능해서, “액션 블록” 개념 연습에 딱 좋아요.

## 5. 몽골 야영지 – 스텔스/전투 레벨 구조 역기획

고스트 오브 쓰시마의 몽골 야영지는
**여러 진입 루트, 잠입/정면 돌파 선택, 인질 구출, 목표 파괴** 등 복합적인 플레이를 허용하는 미니 전장 구조예요.

### 5-1. 게임 안에서 할 관찰

하나의 작고 단순한 야영지를 골라서:

1. **톱다운 맵 그리기**

   * 외곽 울타리, 출입구, 망루, 천막, 포로 위치, 화약통 등 기물 배치.
   * “플레이어가 선택할 수 있는 루트”를 화살표로 표시

     * 정문 돌파
     * 측면 펜스 넘어가기
     * 지붕/높은 곳 접근 등

2. **시야/은신 구조**

   * 경비병의 순찰 루트, 시야가 겹치는 구역, 풀숲 위치 체크.
   * “여길 지나가면 들킬 확률이 높은가? 낮은가?”를 구분.

3. **목표 구조**

   * 인질 구출, 깃발 불태우기, 자원 파괴 등
     야영지 미션에서 요구하는 목표가 레벨 어디에 어떻게 배치돼 있는지 기록.

### 5-2. UE에서 리빌드 과제

* 30m × 30m 정도의 작은 “캠프” 블록아웃 만들기.
* `BP_EnemyGuard`

  * 간단한 시야 콘(Forward Vector + SphereTrace 등)
  * 발견 시 경보 사운드 + 상태 변수 변경.
* `BP_CampObjective`

  * 상호작용하면 “목표 1/3 완료” 같은 UI 표시.
* 최소 2개 이상의 진입 루트를 설계:

  * 높은 곳에서 매달려 들어가는 루트
  * 풀숲을 연결해서 은신 이동하는 루트

이 연습은 **“전투/잠입 레벨의 공간 구조” + “적/목표 액션 블록”**을 동시에 연습하는 셈이라,
레벨 디자이너 + 스크립터 마인드를 함께 키워줘요.

## 6. Tales of Tsushima – 스토리 퀘스트 1개를 완전 분해

Tales of Tsushima는 동료나 NPC의 서사에 초점을 맞춘 사이드 퀘스트 묶음이에요.

하나를 골라서(예: 이시카와, 마사코, 유나의 첫 번째 이야기):

### 6-1. 스토리 구조 분석

* **Act 1 / 2 / 3**로 나눠서:

  * “목표가 뭐로 바뀌는지”
  * “공간이 어떻게 이동하는지(마을 → 길 → 목표 지역 등)”
  * “새로운 정보/반전이 어디서 나오는지”를 정리.

### 6-2. 레벨·액션 블록 관점으로 재정의

예를 들어,

* Act 1: 마을에서 NPC와 대화 → 흔적 추적
* Act 2: 적 추격, 잠입/전투 → 목표 지점 도달
* Act 3: 보스/미니 보스와 전투 + 컷신

이걸 UE에서:

* `BP_NPCDialogue` (대화 후 퀘스트 시작 트리거)
* `BP_InvestigationClue` (근처에 가면 힌트/텍스트 표시)
* `BP_AmbushEncounter` (트리거 존 진입 시 적 스폰)
* `BP_BossArena` (전투 시작 시 카메라/음악/연출 변경)

처럼 “퀘스트 진행용 액션 블록”으로 변환해보는 거죠.

## 7. 역기획 템플릿 예시

나중에 고스트 말고 다른 게임 역기획할 때도 쓸 수 있게,
쓰시마 기준 템플릿을 하나 정해두면 좋아요.

### 예시 – 폭스 덴 템플릿

1. **기본 정보**

   * 콘텐츠 이름: 폭스 덴
   * 카테고리: 탐험 / 미니 퀘스트 / 수집 요소
   * 지속 시간: 2–5분

2. **플레이 루프**

   1. 여우 발견
   2. 여우 따라가기
   3. 신사 도착, 기도
   4. 보상 획득

3. **플레이어 액션 목록**

   * 탐색, 추적, 점프/이동, 상호작용, UI 확인

4. **레벨 요소**

   * 여우 스폰 포인트
   * 안내 동선(지형, 시야 유도, 장애물)
   * 신사 위치(랜드마크, 고지대 등)

5. **액션 블록**

   * FoxGuide, Shrine, Reward, UIFeedback …

6. **UE 구현 계획**

   * 필요한 블루프린트 클래스, 변수, 이벤트, 테스트 플로우

이 템플릿을 **대나무 도장 / 야영지 / Tales**에도 그대로 적용해보면,
“이 게임 전체가 어떤 액션 블록 세트로 구성돼 있는지”가 눈에 보이기 시작합니다.
