이 영상은 EA Maxis의 크리에이티브 디렉터인 Stone Le Brandy가 게임 개발 과정에서 '한 페이지 디자인 문서(One-Page Design Document)'의 중요성과 활용 방법을 설명하는 강연입니다.

전통적인 방식의 두꺼운 '디자인 바이블'이나 관리하기 힘든 위키(Wiki) 기반 문서의 한계를 극복하기 위해, **핵심 정보를 시각화하여 한 페이지에 압축**하는 방식의 장점과 실전 노하우를 다루고 있습니다.

주요 내용을 요약해 드립니다.

### 1. 왜 '한 페이지 디자인'인가?

* **소통의 효율성:** 두꺼운 문서는 아무도 읽지 않지만, 한 페이지 디자인은 시각적으로 매력적이라 동료들이 먼저 관심을 갖고 벽에 붙여두거나 공유하게 만듭니다.
* **설계 과정 그 자체가 디자인:** 문서를 작성하는 과정에서 막연한 아이디어를 구체화하고, 논리적인 모순을 발견하며 명확한 해결책을 찾게 됩니다.
* **복잡성 제거:** 제한된 공간에 정보를 담으려다 보니, 가장 중요한 것이 무엇인지 우선순위를 정하게 되고(Subtractive art), 시스템 간의 관계를 한눈에 파악할 수 있습니다.

### 2. 주요 실전 팁

* **Adobe Illustrator 활용:** 벡터 기반 프로그램을 사용하면 정보를 확대/축소하고 수정하기가 매우 용이하며, 대형 출력물(플로터 사용)로 인쇄했을 때 가독성이 압도적으로 좋습니다.
* **시각적 요소 활용:** 텍스트만 나열하지 말고, 중앙에 메인 일러스트를 배치하고 주변에 설명(Call-out)을 다는 방식을 권장합니다.
* **시간과 흐름 포함:** 게임 디자인은 시간의 흐름(레벨, 퀘스트 등)이 중요하므로, 타임라인이나 흐름도(Flowchart)를 활용해 플레이어가 경험하게 될 여정을 표현하세요.
* **직접 인쇄하여 배포:** 파일을 이메일로 보내지 말고, 직접 인쇄해서 팀원에게 전달하고, 그 문서 위에 직접 메모하고 수정하면서 소통하십시오. 이것이 살아있는 협업 방식입니다.

### 3. 문서 작성 프로세스 (예시: 스포어)

* **매트릭스(Matrix) 활용:** 복잡한 시스템을 해결할 때 엑셀 등으로 매트릭스를 먼저 만들고 '하향식(Top-down)'으로 정보를 채워 넣으면, 빠진 부분 없이 모든 경우의 수를 완벽하게 설계할 수 있습니다.
* **반복과 수정:** 처음부터 완벽할 필요는 없습니다. 거친 초안을 만들고 동료들의 의견을 받아 수정을 거듭하며 정보를 더 깔끔하고 명확하게 다듬어 나갑니다.

### 4. 강연의 핵심 메시지

디자인 문서는 단순히 정보를 기록하는 도구가 아니라, **아이디어를 팀원들과 공유하고 모두가 같은 비전을 보게 만드는 가장 강력한 소통 수단**입니다. "어떻게 하면 더 많은 정보를 한 페이지에 아름답고 명확하게 담을 수 있을까?"를 끊임없이 고민할 때 더 나은 게임 디자인이 탄생합니다.

# 오리지널리티를 만드는 7단계 로드맵

## 1단계. “우리는 어떤 액션 감정을 팔 것인가?”부터 정하기

처음부터 “새로운 시스템”을 찾으면 막힙니다. 먼저 플레이어가 느낄 감정을 정해야 합니다.

PC 액션 게임이라면 예를 들어 이런 감정 축이 있습니다.

| 감정 목표 | 게임 방향                 |
| ----- | --------------------- |
| 압도감   | 적을 쓸어버리는 파워 판타지       |
| 긴장감   | 한 번의 실수가 치명적인 하드코어 액션 |
| 유려함   | 끊기지 않는 콤보와 이동         |
| 무게감   | 공격 하나하나가 묵직한 전투       |
| 즉흥성   | 상황에 따라 무기를 바꾸는 전투     |
| 숙련감   | 플레이어 실력이 눈에 보이게 상승    |
| 기괴함   | 낯선 생물, 이상한 룰, 불안한 세계  |

MDA 프레임워크는 게임을 **Mechanics, Dynamics, Aesthetics**로 나누어 봅니다. 개발자는 규칙과 시스템을 만들지만, 플레이어가 실제로 소비하는 것은 감정과 경험이라는 관점입니다. 즉 “우리만의 오리지널리티”는 시스템 이름이 아니라 **플레이어가 느끼는 고유한 감정 조합**이어야 합니다. ([노스웨스턴 컴퓨터 과학][1])

추천 질문은 이겁니다.

> 우리 게임을 한 문장으로 말하면, 플레이어는 어떤 방식으로 강해지고, 어떤 방식으로 위험해지며, 어떤 방식으로 짜릿해야 하는가?

예시:

> “플레이어는 적의 공격을 피하는 게 아니라, 적의 공격을 일부러 받아내며 리듬을 빼앗는 액션 게임.”

이 정도까지 감정과 행동이 붙어야 오리지널리티의 씨앗이 됩니다.

## 2단계. 경쟁작을 “장르”가 아니라 “전투 문법”으로 분해하기

“소울라이크”, “데빌 메이 크라이 스타일”, “하데스 느낌”처럼 장르명으로 비교하면 차별화가 안 됩니다. 대신 액션 게임을 아래처럼 분해해야 합니다.

| 분석 항목 | 봐야 할 것                    |
| ----- | ------------------------- |
| 입력    | 버튼 수, 입력 타이밍, 차지/홀드/연타 여부 |
| 이동    | 회피, 대시, 점프, 공중 제어, 벽 이동   |
| 공격    | 콤보, 캔슬, 경직, 넉백, 범위        |
| 방어    | 회피, 패링, 가드, 슈퍼아머, 반격      |
| 적 설계  | 공격 예고, 패턴 길이, 무리 구성       |
| 카메라   | 고정, 락온, 자유 시점, 근접감        |
| 리듬    | 빠른 교전, 느린 심리전, 웨이브 전투     |
| 보상    | 숙련 보상, 성장 보상, 파밍 보상       |

여기서 목표는 “비슷한 게임 찾기”가 아니라 **우리가 건드릴 수 있는 빈칸 찾기**입니다.

예를 들어 경쟁작들이 대부분 “회피 후 반격” 구조라면, 여러분은 다음처럼 틀 수 있습니다.

| 흔한 구조      | 변형 방향                |
| ---------- | -------------------- |
| 회피 후 반격    | 공격을 흘려보내며 위치를 바꾸는 전투 |
| 패링 성공 시 반격 | 패링 성공 시 적의 기술을 훔침    |
| 콤보 누적      | 콤보를 오래 유지할수록 세계가 변형됨 |
| 무기 교체      | 무기가 아니라 신체 부위를 교체    |
| 스태미나 관리    | 체력이 곧 공격 자원          |

PC 시장은 인디와 중소 규모 게임도 강한 기회가 있지만, 동시에 Steam 내 경쟁과 노출 문제도 큽니다. 2025년 Steam에서 인디 게임이 상당한 매출 비중을 차지했다는 분석이 있는 반면, 출시작 과잉과 가시성 경쟁도 계속 언급되고 있습니다. 따라서 오리지널리티는 “개발팀 내부 만족”이 아니라 **상점 페이지 GIF 3초 안에 보이는 차별점**이어야 합니다. ([Notebookcheck][2])

## 3단계. “오리지널리티 후보”를 3종류로 나누기

오리지널리티는 한 종류가 아닙니다. 보통 세 층으로 나뉩니다.

## A. 조작 오리지널리티

플레이어의 손맛 자체가 다른 경우입니다.

예시:

* 공격 버튼을 누르는 게 아니라, 마우스 방향으로 칼의 궤적을 직접 긋는다.
* 회피가 이동기가 아니라 공격 준비 동작이다.
* 콤보가 버튼 순서가 아니라 박자감으로 만들어진다.
* 적을 때리는 것이 아니라 지형에 부딪히게 만든다.

PC 액션이면 키보드+마우스, 패드 양쪽을 고려해야 하므로 조작 오리지널리티는 강력하지만 위험도 큽니다. 패드와 키마 모두에서 재미있게 작동하는지 초기에 확인해야 합니다.

## B. 전투 규칙 오리지널리티

시스템의 룰이 다른 경우입니다.

예시:

* 적을 죽이면 강해지는 게 아니라, 적을 살려둘수록 전투장이 유리하게 변한다.
* 체력 회복이 공격 성공이 아니라 위험한 위치 선점으로 이루어진다.
* 무기가 장비가 아니라 적의 패턴을 일시적으로 빌리는 방식이다.
* 보스가 체력바를 잃는 게 아니라 기억, 시야, 공간을 잃는다.

## C. 표현 오리지널리티

보이는 톤, 세계관, 애니메이션, 사운드가 다른 경우입니다.

예시:

* 모든 공격이 붓질처럼 보이는 수묵 액션
* 적의 공격이 음악 악보처럼 시각화되는 리듬 액션
* 폐공장 산업 사운드로만 구성된 전투 피드백
* 캐릭터가 검을 휘두르는 게 아니라 그림자를 접고 펴며 싸움



추천 구조:

> 전투 규칙 1개를 새롭게 만든다.
> 조작은 익숙하게 둔다.
> 표현은 그 규칙이 잘 보이도록 과감하게 통일한다.

## 4단계. “오리지널리티 원형”을 2주 안에 검증하기



목표는 예쁜 데모가 아니라 아래 5가지를 확인하는 것입니다.

| 검증 항목  | 질문                           |
| ------ | ---------------------------- |
| 10초 재미 | 아무 설명 없이 10초 안에 재미가 느껴지는가?   |
| 반복성    | 같은 전투를 20번 해도 조금씩 다르게 느껴지는가? |
| 숙련 곡선  | 잘하는 사람과 못하는 사람의 차이가 보이는가?    |
| 시각 전달  | GIF만 봐도 차별점이 보이는가?           |
| 확장성    | 적, 무기, 스테이지로 6개월 이상 확장 가능한가? |

액션 게임에서 특히 중요한 것은 **게임 필**입니다. Pichlmair와 Johansen의 게임 필 연구는 순간적인 상호작용의 감각을 물리성, 증폭, 지원의 관점으로 설명합니다. 쉽게 말하면 이동의 예측 가능성, 히트스톱·카메라 셰이크·사운드 같은 피드백, 플레이어 의도를 잘 받아주는 입력 보정이 액션의 감각을 좌우합니다. ([arXiv][3])

이 단계의 산출물은 이런 식이어야 합니다.

> “우리 게임은 패링 액션이다”가 아니라
> “적의 공격을 정확히 막으면 반격하는 게임”도 아니라
> “적의 공격 방향을 흡수해서 내 다음 공격 궤적으로 바꾸는 게임”이다.

이렇게 말했을 때 시스템, 조작, 이펙트, 적 설계가 한 방향으로 묶이면 오리지널리티가 생기기 시작합니다.

## 5단계. “세로 절편”으로 팀 전체 파이프라인을 검증하기

Core Prototype이 재미있다면 다음은 **Vertical Slice**입니다.

구성은 이 정도면 충분합니다.

| 구성 요소        | 권장 범위            |
| ------------ | ---------------- |
| 플레이어 캐릭터     | 1명               |
| 핵심 무기/전투 스타일 | 1개               |
| 일반 적         | 3종               |
| 엘리트 적        | 1종               |
| 보스 또는 미니보스   | 1종               |
| 스테이지         | 1개               |
| 성장/보상        | 최소 버전            |
| UI/사운드/피드백   | 실제 출시 품질의 60~70% |

이 단계의 목적은 “많이 만들기”가 아니라 **우리만의 재미가 콘텐츠로 확장되는지** 확인하는 것입니다.

예를 들어 핵심이 “적의 공격을 흡수해 내 기술로 변환”이라면:

* 일반 적 A는 직선 공격을 준다.
* 일반 적 B는 곡선 공격을 준다.
* 엘리트 적은 공격 속성을 섞는다.
* 보스는 플레이어가 흡수한 기술을 다시 카운터친다.
* 스테이지 기믹은 흡수한 공격으로만 열 수 있다.

이렇게 전투, 적, 보스, 레벨, 성장 구조가 하나의 코어에서 파생되어야 합니다.

## 6단계. “오리지널리티 헌장”을 문서화하기


문서 이름은 거창하지 않아도 됩니다.
**Originality Pillars 1-page** 정도면 충분합니다.

구성은 이렇게 추천합니다.

## Originality Pillars

### 1. 우리 게임의 한 문장

> 플레이어는 [핵심 행동]을 통해 [위험]을 다루고, [감정]을 느낀다.

### 2. 절대 지킬 3가지

예:

1. 플레이어는 항상 공격적으로 문제를 해결한다.
2. 모든 방어 행동은 다음 공격으로 연결된다.
3. 적의 패턴은 읽기 쉽지만, 활용 방식은 깊어야 한다.

### 3. 하지 않을 것 5가지

예:

1. 의미 없는 스탯 파밍을 넣지 않는다.
2. 긴 컷신으로 액션 리듬을 끊지 않는다.
3. 플레이어를 즉사 함정으로 놀라게 하지 않는다.
4. 모든 무기를 비슷한 DPS 구조로 만들지 않는다.
5. 카메라가 전투 판단을 방해하지 않게 한다.

### 4. GIF로 보여야 할 차별점

> 상점 페이지, 트레일러, SNS 클립에서 3초 안에 보여야 하는 장면은 무엇인가?

이 항목이 중요합니다. PC 액션 게임은 영상으로 팔리는 경우가 많습니다. 오리지널리티가 설명문을 읽어야만 이해된다면 시장에서 약합니다.

## 7단계. 플레이테스트로 “새로움”이 아니라 “기억나는 것”을 측정하기

테스트할 때 “재미있었나요?”만 물으면 안 됩니다. 오리지널리티는 기억에 남아야 합니다.

플레이테스트 질문은 이렇게 바꾸세요.

| 나쁜 질문      | 좋은 질문                        |
| ---------- | ---------------------------- |

특히 액션 게임은 “다시 하면 더 잘할 수 있을 것 같다”는 감각이 중요합니다. 이게 있으면 숙련 기반 오리지널리티가 살아 있는 것입니다.

# 실전 워크숍: 2주짜리 오리지널리티 스프린트

바로 실행한다면 이렇게 추천합니다.

## Day 1: 기준 정하기

팀 전체가 아래 문장을 각각 5개씩 씁니다.

> 우리 게임은 플레이어가 [행동]할 때 가장 짜릿해야 한다.

예:

* 적의 공격을 빼앗을 때
* 공중에서 착지하지 않고 계속 싸울 때
* 위험한 타이밍에 앞으로 파고들 때
* 무기를 버릴수록 강해질 때
* 적을 직접 때리지 않고 서로 충돌시킬 때

## Day 2: 후보 10개 뽑기

각 아이디어를 아래 기준으로 점수화합니다.

| 기준           |  점수 |
| ------------ | --: |
| 10초 안에 이해 가능 | 1~5 |
| 액션 손맛 가능성    | 1~5 |
| 영상 차별성       | 1~5 |
| 콘텐츠 확장성      | 1~5 |
| 팀 현실성    | 1~5 |

상위 3개만 남깁니다.

## Day 3~7: 프로토타입 3개 제작

각 후보는 아주 작게 만듭니다.

* 회색 박스 맵
* 플레이어 기본 캐릭터
* 적 1종
* 공격/회피/피드백
* 성공/실패 조건

그래픽 품질은 버리고, 입력감과 전투 결과만 봅니다.

## Day 8: 내부 테스트

팀원이 아닌 사람 5~10명에게 플레이시킵니다.
설명은 최소화합니다.

물어볼 것은 3개입니다.

1. 가장 기억나는 순간은?
2. 다른 게임과 다르게 느껴진 점은?
3. 다시 하면 더 잘할 수 있을 것 같은가?

## Day 9~12: 하나만 선택해서 강화

가장 반응이 좋은 후보 하나를 골라 다음을 추가합니다.

* 적 2종 추가
* 전투 공간 2개 추가
* 피드백 강화
* 짧은 보상 루프 추가
* 30초짜리 클립 제작

## Day 13~14: 오리지널리티 헌장 작성

마지막에 1페이지로 정리합니다.

> 이 게임은 무엇인가?
> 무엇이 다른가?
> 무엇은 절대 하지 않는가?
> 어떤 장면이 반드시 트레일러에 들어가야 하는가?

# 개인적으로 추천하는 방향

현재 조건이 **PC / 액션 / 15명 팀**이라면, 오리지널리티를 “대규모 콘텐츠”에서 찾기보다 아래 중 하나에서 찾는 게 좋습니다.

## 1. 전투 리듬

예: 회피-공격이 아니라, 흡수-변환-폭발 구조.

장점: 영상으로 잘 보임.
위험: 밸런스와 피드백 설계가 어렵다.

## 2. 이동 액션

예: 벽, 공중, 적의 몸, 무기 투척 지점을 모두 이동 자원으로 사용.

장점: PC 액션에서 손맛이 강함.
위험: 카메라와 레벨 디자인 부담이 크다.

## 3. 적과 상호작용하는 방식

예: 적을 처치하는 게 아니라, 적의 패턴을 조합해 전장을 푸는 액션.

장점: 전투와 퍼즐성이 섞여 차별화 가능.
위험: 너무 복잡하면 액션 쾌감이 죽는다.

## 4. 위험을 보상으로 바꾸는 구조

예: 피격 직전까지 접근할수록 공격력, 시야, 시간 조작 게이지가 증가.

장점: 숙련자 플레이가 멋있게 보임.
위험: 초보자가 불공정하게 느낄 수 있다.

# 핵심 결론

여러분 팀이 밟아야 할 단계는 이 순서가 좋습니다.

1. **플레이어 감정 정의**
   “우리 게임은 어떤 액션 감정을 주는가?”

2. **전투 문법 분해**
   경쟁작을 장르가 아니라 입력, 이동, 공격, 방어, 적 설계로 나눈다.

3. **차별화 후보 3개 제작**
   말로 토론하지 말고 작은 프로토타입으로 만든다.

4. **10초 재미 검증**
   설명 없이 해도 손맛과 차별점이 느껴지는지 본다.

5. **Vertical Slice 제작**
   적, 보스, 레벨, 보상까지 하나의 코어에서 확장되는지 확인한다.

6. **Originality Pillars 문서화**
   팀 전체가 같은 기준으로 “우리 게임답다”를 판단하게 한다.

7. **기억나는 순간 측정**
   플레이어가 끝나고 말하는 한 장면이 곧 여러분의 오리지널리티다.

가장 중요한 문장은 이겁니다.

> 오리지널리티는 “남들이 안 한 것”이 아니라, **우리 게임에서만 반복하고 싶어지는 행동**입니다.
