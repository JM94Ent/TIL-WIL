# 1. 📄 액션 기반 규격 문서 구조 (템플릿)

## ① 캐릭터 기본 스펙

| 항목       | 값     | 비고     |
| -------- | ----- | ------ |
| 이동 속도    | 5 m/s | 기준 단위  |
| 대쉬 거리    | 6 m   | 0.4초   |
| 점프 높이    | 2.5 m | 최대     |
| 점프 거리    | 4 m   | 전진 기준  |
| 낙하 안전 높이 | 5 m   | 데미지 없음 |
| 카메라 거리   | 6~8 m | 3인칭 기준 |

👉 여기서 중요한 것
→ “이 값이 아니라 **비율**이 핵심입니다”

---

## ② 전투 관련 규격

| 항목         | 값     | 설계 의도    |
| ---------- | ----- | -------- |
| 기본 공격 사거리  | 2.5 m | 근접 기준    |
| 강공격 사거리    | 3.5 m | 리치 확보    |
| 회피 거리      | 6 m   | 대쉬와 동일   |
| i-frame    | 0.3초  | 회피 성공 판정 |
| 적 기본 공격 범위 | 5 m   | 간신히 피하게  |

👉 핵심 구조

* 적 공격 > 플레이어 기본 이동
* 플레이어 대쉬 > 적 공격 범위

---

## ③ 공간 규격 (레벨용)

| 요소           | 기준            |
| ------------ | ------------- |
| 플랫폼 간 거리     | 3m / 4m / 6m  |
| 벽 간 점프 가능 거리 | 4m            |
| 복도 폭         | 최소 3m / 전투 6m |
| 전투 공간        | 최소 10m x 10m  |
| 보스 공간        | 20m 이상        |

# 2. 🧱 블록아웃 테스트맵 구성

👉 핵심: “모든 액션을 수치로 검증하는 공간”

---

## ① 이동 테스트 존

구성:

* 2m / 4m / 6m / 8m 간격 플랫폼
* 점프 높이별 계단 구조

테스트 목적:

* 점프 성공률
* 거리 체감
* 낙하 스트레스

---

## ② 대쉬 / 회피 테스트 존

구성:

* 일정 속도로 날아오는 더미 공격 (직선/원형)
* 거리 4m / 5m / 6m / 7m 구간

테스트:

* “언제 맞고 언제 피하는지”

👉 여기서 i-frame 조정합니다

---

## ③ 전투 거리 테스트 존

구성:

* 더미 적 (고정)
* 거리 2m / 3m / 4m / 5m 배치

테스트:

* 공격이 닿는 거리
* 콤보 유지 거리
* 적 밀림 거리 (넉백)

---

## ④ 다수 전투 공간 테스트

구성:

* 좁은 방 (6x6)
* 중간 방 (10x10)
* 넓은 방 (15x15)

테스트:

* 카메라 답답함
* 회피 가능 여부
* 적 수용 한계

---

## ⑤ 지형 활용 테스트 (세키로식)

참고:

* 세키로: 섀도우 다이 트와이스

구성:

* 고저차 2m / 4m / 6m
* 벽 점프 / 그래플 포인트

테스트:

* 수직 이동 자연스러움
* 카메라 충돌

---

# 3. 📏 규격 설계 핵심 공식 (진짜 중요)

## 기준 단위 정의

👉 “1 Unit = 플레이어 1초 이동 거리”

예:

* 이동속도 5m/s
  → 1 Unit = 5m

---

## 규격 만들 때 공식

* 점프 거리 = 0.8 ~ 1.2 Unit
* 대쉬 거리 = 1 ~ 1.5 Unit
* 적 공격 범위 = 0.8 ~ 1 Unit
* 전투 공간 = 최소 2~3 Unit

👉 이렇게 하면 밸런스 자동으로 맞습니다

---

# 4. 🎮 레퍼런스 적용 방식

## 니어 오토마타

* 거리 여유 있음 → 회피 중심

## 데빌 메이 크라이 5

* 거리 타이트 → 콤보 유지

## 스텔라 블레이드

* 타이밍 기반 → 패링 중심

👉 정민님 게임 방향에 따라
“거리형 / 타이밍형 / 콤보형” 중 하나를 선택해야 합니다
